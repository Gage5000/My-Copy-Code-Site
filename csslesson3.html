CSS Positioning

In CSS, positioning is a big part of the styling process and by using overflow, position, display, and z-index properties you can change that.

CSS Position

The position property specifies the type of positioning method used for an element that being static, relative, fixed, absolute or sticky.

  • HTML elements are positioned static by default and static positioned elements are not affected by the top, bottom, left, and right properties.
  • A relative element is positioned relative to its normal position.
  • A fixed element is positioned relative to the viewport, meaning it always stays in the same place even if the page is scrolled. The top, right, bottom, and left properties are used to position an fixed element.
  • A absolute element is positioned relative to the nearest positioned ancestor instead of positioned relative to the viewport, like fixed.
  • A sticky element toggles between relative and fixed, depending on the scroll position. It is positioned relative until a given offset position is met in the viewport then it sticks in place like it did with position fixed.

h1 {

position: static;

}

h2 {

position: relative;

}

h3 {

position: fixed;

}

h4 {

position: absolute;

}

h5 {

position: sticky;

}

CSS Overflow

The CSS overflow property controls what happens to content that is too big to fit into an area.

Here are all of the overflow opptions:

  • visible - The default. The overflow is not clipped but the content renders outside the element's box
  • hidden - The overflow is clipped, and the rest of the content will be invisible
  • scroll - The overflow is clipped, and a scrollbar is added to see the rest of the content
  • auto - Similar to scroll, but it adds scrollbars only when necessary

.visible {

background-color: green;

overflow: visible;

}

.hidden {

background-color: red;

overflow: hidden;

}

.scroll {

background-color: violet;

overflow: scroll;

}

.auto {

background-color: goldenrod;

overflow: auto;

}

CSS Display

The display property is used to specify how an element is shown on a web page.

The default display value for most elements is block or inline.

Here are some of the most used displays

  • inline - Displays an element as an inline element
  • block - Displays an element as a block element
  • flex - Displays an element as a block-level flex container
  • grid - Displays an element as a block-level grid container
  • inline-block - Displays an element as an inline-level block container. The element itself is formatted as an inline element, but you can apply height and width values
  • inline-flex - Displays an element as an inline-level flex container
  • inline-grid - Displays an element as an inline-level grid container

.inline{

display: inline;

background-color: red;

}

.block {

display: block;

background-color: green;

}

.flex {

display: flex;

background-color: palevioletred;

}

.grid {

display: grid;

background-color: cornflowerblue;

}

.inline-block {

display: inline-block;

background-color: lightgreen;

}

.inline-flex {

display: inline-flex;

background-color: blueviolet;

}

.inline-grid {

display: inline-grid;

background-color: saddlebrown;

}

CSS Z-index

When elements are positioned, they can overlap other elements.

The z-index property specifies the stack order of an element and it can have a positive or negative stack order like -1 or 1.


div {

position: absolute; z-index: -1; background-color: red; right: 0px; left: 0px;

}
