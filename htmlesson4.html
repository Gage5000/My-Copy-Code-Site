Div and Semantic Tags

Semantic tags clearly describe its meaning to the browser and the developer.

For example non-semantic elements like <div> and <span> tells nothing about its content but semantic tags like <nav> and <article> clearly defines its content.

The Div Tag

The <div> tag is used to group other tag to make a section and it takes all available width, and comes with line breaks before and after meaning it is a block tag.

Becuase it is good at grouping other tags it is used alot for making webpages layouts.

The Header Tag

The <header> tag defines a container for introductory content or navigational links..

A header typically is used to contains one or more heading elements, logos, icons, or, authorship information.

You can have several header tags in one HTML document but the header tag cannot be placed within a footer, address, or another header tag.

The Figure and Figcaption Tags

The <figure> tag specifies self-contained content, like illustrations, diagrams, photos, code listings, etc.

The <figcaption> tag defines a caption for a figure tag. The figcaption tag can be placed as the first or as the last child of a figure tag.

The img element defines the actual image/illustration not the content within the figure.


<figure>

<img src="REPLACE THIS WITH YOUR LINK">

<figcaption>Fig.1 - By: bytecodeminer.</figcaption>

</figure>

The Summary and Details Tags

The <details> tag defines additional details that the user can open and close on demand.

The details tag is often used to create an interactive widget that the user can open and close. By default, the widget is closed. When open, it expands, and displays the content within.

The <summary> tag is used in conjunction with details to specify a visible heading for the details.


<details>

<summary>This is the 1st div</summary>

<p>Paragraph Text one</p>

<p>Paragraph Text two</p>

</details>
summary tag

This is the summary tag and as you can see you can summary this text within the details tag.
